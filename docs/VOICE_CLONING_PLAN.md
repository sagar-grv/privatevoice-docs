# Voice Cloning Plan

Personal voice work begins only after typed offline RAG and standard offline TTS are reliable.

1. Consent, live random-phrase verification, recording guidance, quality checks, profile storage, and a clearly labelled unavailable/mock engine.
2. Desktop-assisted evaluation of candidate architectures without making desktop/cloud processing part of production.
3. Export a compact quantized speaker/profile model for a supported Android runtime.
4. Prove fully offline synthesis, consent ownership, language quality, latency, memory, storage, and deletion.

Voice identity and speaking style remain separate models. No arbitrary uploaded-audio cloning, background microphone use, or silent third-party cloning is permitted.
